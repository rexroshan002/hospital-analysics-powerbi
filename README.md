# 📊 Project Overview

This project delivers a **Business Intelligence analytics system** designed to transform fragmented hospital operational data into **clear financial and operational insights**.

Healthcare datasets are often inconsistent, messy, and difficult to analyze directly.  
This solution focuses on building a **structured analytics layer** capable of supporting **decision-making at both executive and operational levels**.

The dashboard provides visibility into:

• Revenue performance  
• Patient behavior patterns  
• Doctor productivity  
• Service profitability  
• Hospital operational workload  

The system demonstrates the **end-to-end BI workflow**, including data cleaning, modeling, transformation, and visualization.

---

# 🎯 Business Objectives

The analytics system answers several key hospital management questions:

**Financial Insights**
- What is the total revenue generated by the hospital?
- Which services contribute the most revenue?
- Which doctors generate the highest value per visit?

**Operational Insights**
- Which days experience the highest patient traffic?
- What is the patient retention rate?
- How does workload distribute across providers?

**Strategic Insights**
- Which demographic groups generate the most revenue?
- Which acquisition sources bring the most valuable patients?
- How can staffing be optimized based on visit patterns?

# 🏗 System Architecture

The analytical model was designed using a **Star Schema architecture**, which is widely used in production-grade Business Intelligence systems due to its performance and scalability.

### Fact Table
**visits**

Contains transactional records for every hospital visit including:
- patient
- doctor
- service
- visit type
- visit date
- revenue generated

### Dimension Tables

**patients**  
Stores patient demographics and acquisition sources.

**doctors**  
Defines provider identities and specialization hierarchy.

**services**  
Categorizes medical procedures and consultation types.

**date_table**  
A generated calendar dimension used for time-based analytics and chronological reporting.

This architecture ensures:

- fast aggregations
- simplified filtering
- clear relationship paths
- scalable analytics logic

# 🧹 Data Engineering & Cleaning

Real-world healthcare datasets rarely arrive in clean formats.  
This project includes extensive data preparation to ensure reliable analysis.

Key transformations performed:

### Date Standardization
Multiple date formats were detected across the dataset.  
A defensive parsing strategy was implemented to safely interpret:

- `MM/DD/YYYY`
- `DD-MM-YYYY`
- `YYYY/MM/DD`

This prevents refresh failures caused by inconsistent formats.

---

### Data Normalization

Categorical inconsistencies were corrected including:

| Issue | Example | Resolution |
|-----|-----|-----|
| Gender variations | Male / male / M | Standardized to single value |
| Trailing whitespace | "Referral " | Trimmed |
| Service categories | Fragmented labels | Normalized categories |

---

### Data Quality Improvements

Additional safeguards included:

- duplicate record detection
- null value handling
- categorical consolidation
- column type enforcement

These steps ensure the analytics layer reflects **accurate business reality** rather than raw data noise.
